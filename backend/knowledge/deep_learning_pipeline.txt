# Deep Learning Pipeline

## Overview

Deep Learning is a subset of machine learning that uses neural networks with multiple layers (hence "deep") to learn hierarchical representations of data. Unlike traditional machine learning, deep learning automatically discovers features from raw data, making it particularly powerful for complex tasks like image recognition, natural language processing, and speech recognition.

## When to Use Deep Learning

### Decision Tree: Should You Use Deep Learning?

```
Start
  |
  v
Do you have large dataset? (>10,000 samples)
  |
  +-- No --> Consider traditional ML (Random Forest, XGBoost)
  |
  v
Do you have unstructured data? (images, text, audio)
  |
  +-- Yes --> Deep Learning is recommended
  |
  v
Is computational resources available? (GPU preferred)
  |
  +-- No --> Consider lighter models or transfer learning
  |
  v
Deep Learning is appropriate
```

### When to Use Deep Learning vs Traditional ML

| Scenario | Recommended Approach |
|----------|---------------------|
| Tabular data, <10K samples | XGBoost, LightGBM |
| Tabular data, >10K samples | Deep Learning viable |
| Images | CNN, Transfer Learning |
| Text/Sequences | RNN, LSTM, Transformer |
| Time Series (short) | XGBoost with lag features |
| Time Series (long) | LSTM, Transformer |
| Audio/Speech | CNN, RNN, Transformers |
| High-dimensional data | Autoencoders |

### Dataset Size Guidelines

| Dataset Size | Recommendation |
|--------------|---------------|
| < 1,000 | Traditional ML or Transfer Learning |
| 1,000 - 10,000 | Transfer Learning or small NN |
| 10,000 - 100,000 | Deep Learning viable |
| > 100,000 | Deep Learning recommended |
| > 1,000,000 | Deep Learning essential |

### 1. Feedforward Neural Network (MLP/DNN)

**Best for**: Tabular data, classification, regression

**When to use**:
- Structured/tabular data with many features
- When interpretability is less critical
- As part of larger ensemble models

**Theoretical Background**:
- Universal approximator - can approximate any continuous function
- Input → Hidden Layers → Output
- Each neuron: weighted sum + bias + activation

```python
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Dense, Dropout, BatchNormalization, Input
from tensorflow.keras.optimizers import Adam

def build_mlp(input_dim, num_classes=1, hidden_units=[256, 128, 64], dropout_rate=0.3):
    """
    Build a Feedforward Neural Network for classification or regression
    
    Args:
        input_dim: Number of input features
        num_classes: 1 for regression/binary, >1 for multiclass
        hidden_units: List of units in each hidden layer
        dropout_rate: Dropout rate for regularization
    """
    model = Sequential()
    model.add(Input(shape=(input_dim,)))
    
    for units in hidden_units:
        model.add(Dense(units, activation='relu'))
        model.add(BatchNormalization())
        model.add(Dropout(dropout_rate))
    
    # Output layer
    if num_classes == 1:
        model.add(Dense(1, activation='sigmoid' if binary else 'linear'))  # binary classification or regression
    else:
        model.add(Dense(num_classes, activation='softmax'))  # multiclass
    
    return model

# Binary classification example
model = build_mlp(input_dim=100, num_classes=1, hidden_units=[256, 128, 64])
model.compile(
    optimizer=Adam(learning_rate=0.001),
    loss='binary_crossentropy',
    metrics=['accuracy']
)

# Multiclass classification example
model = build_mlp(input_dim=100, num_classes=10, hidden_units=[256, 128, 64])
model.compile(
    optimizer=Adam(learning_rate=0.001),
    loss='categorical_crossentropy',
    metrics=['accuracy']
)

# Regression example
model = build_mlp(input_dim=100, num_classes=1, hidden_units=[256, 128, 64])
model.compile(
    optimizer=Adam(learning_rate=0.001),
    loss='mse',
    metrics=['mae']
)
```

### 2. Convolutional Neural Network (CNN)

**Best for**: Image classification, object detection, image segmentation

**When to use**:
- Image data
- Spatial data with local patterns
- When translation invariance is needed

**Theoretical Background**:
- Convolutional layers capture local spatial patterns
- Pooling reduces spatial dimensions
- Filters learn to detect edges, textures, shapes

```python
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import (
    Conv2D, MaxPooling2D, AveragePooling2D,
    Flatten, Dense, Dropout, BatchNormalization,
    Input, GlobalAveragePooling2D
)

def build_cnn(input_shape=(224, 224, 3), num_classes=10, use_batch_norm=True):
    """
    Build a CNN for image classification
    
    Args:
        input_shape: Shape of input images (H, W, C)
        num_classes: Number of output classes
        use_batch_norm: Whether to use batch normalization
    """
    model = Sequential(name='image_classifier')
    model.add(Input(shape=input_shape))
    
    # Block 1: 224 -> 112
    model.add(Conv2D(32, (3, 3), padding='same', activation='relu'))
    if use_batch_norm:
        model.add(BatchNormalization())
    model.add(Conv2D(32, (3, 3), padding='same', activation='relu'))
    if use_batch_norm:
        model.add(BatchNormalization())
    model.add(MaxPooling2D((2, 2)))
    model.add(Dropout(0.25))
    
    # Block 2: 112 -> 56
    model.add(Conv2D(64, (3, 3), padding='same', activation='relu'))
    if use_batch_norm:
        model.add(BatchNormalization())
    model.add(Conv2D(64, (3, 3), padding='same', activation='relu'))
    if use_batch_norm:
        model.add(BatchNormalization())
    model.add(MaxPooling2D((2, 2)))
    model.add(Dropout(0.25))
    
    # Block 3: 56 -> 28
    model.add(Conv2D(128, (3, 3), padding='same', activation='relu'))
    if use_batch_norm:
        model.add(BatchNormalization())
    model.add(Conv2D(128, (3, 3), padding='same', activation='relu'))
    if use_batch_norm:
        model.add(BatchNormalization())
    model.add(MaxPooling2D((2, 2)))
    model.add(Dropout(0.25))
    
    # Classifier
    model.add(GlobalAveragePooling2D())
    model.add(Dense(256, activation='relu'))
    model.add(Dropout(0.5))
    model.add(Dense(num_classes, activation='softmax'))
    
    return model

# Build and compile
model = build_cnn(input_shape=(224, 224, 3), num_classes=1000)
model.compile(
    optimizer='adam',
    loss='categorical_crossentropy',
    metrics=['accuracy']
)
```

### 3. Recurrent Neural Network (RNN/LSTM/GRU)

**Best for**: Sequence data, time series, text data

**When to use**:
- Sequential data where order matters
- Time series forecasting
- Natural language processing
- Video analysis (frame sequences)

**Theoretical Background**:
- Hidden state captures information from previous time steps
- LSTM/GRU solve vanishing gradient problem
- Bidirectional for tasks needing full context

```python
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import (
    LSTM, GRU, Dense, Dropout, Input,
    Bidirectional, Embedding, GlobalMaxPooling1D
)

def build_lstm_sequence_classification(
    vocab_size=10000,
    max_length=200,
    embedding_dim=128,
    lstm_units=128,
    num_classes=2
):
    """Build LSTM for text/sequence classification"""
    model = Sequential([
        Input(shape=(max_length,)),
        Embedding(vocab_size, embedding_dim, input_length=max_length),
        Bidirectional(LSTM(lstm_units, return_sequences=True)),
        Dropout(0.3),
        Bidirectional(LSTM(lstm_units // 2)),
        Dropout(0.3),
        Dense(64, activation='relu'),
        Dropout(0.5),
        Dense(num_classes, activation='softmax')
    ])
    return model

def build_gru_time_series(
    lookback=60,
    n_features=10,
    n_steps_ahead=1
):
    """Build GRU for time series forecasting"""
    model = Sequential([
        Input(shape=(lookback, n_features)),
        GRU(64, return_sequences=True),
        Dropout(0.2),
        GRU(32),
        Dropout(0.2),
        Dense(16, activation='relu'),
        Dense(n_steps_ahead)  # Linear activation for regression
    ])
    return model

# Text classification
text_model = build_lstm_sequence_classification(
    vocab_size=10000,
    max_length=200,
    num_classes=2
)
text_model.compile(
    optimizer='adam',
    loss='categorical_crossentropy',
    metrics=['accuracy']
)

# Time series forecasting
ts_model = build_gru_time_series(
    lookback=60,
    n_features=10,
    n_steps_ahead=1
)
ts_model.compile(
    optimizer='adam',
    loss='mse'
)
```

### 4. Transformer Architecture

**Best for**: NLP, sequence modeling, state-of-the-art performance

**When to use**:
- When best possible accuracy is needed
- Large datasets available
- Transfer learning is desired
- Multiple modalities (text, images, audio)

**Theoretical Background**:
- Self-attention mechanism captures all positions
- Parallel processing (unlike RNNs)
- Scales well with data and compute
- Pre-training + fine-tuning paradigm

```python
from transformers import TFAutoModel, AutoTokenizer
import tensorflow as tf
from tensorflow.keras.layers import Dense, GlobalAveragePooling1D, Input
from tensorflow.keras.models import Model

def build_bert_classifier(
    model_name='bert-base-uncased',
    num_classes=2,
    dropout_rate=0.1
):
    """Build BERT-based text classifier with transfer learning"""
    # Load pre-trained BERT
    bert = TFAutoModel.from_pretrained(model_name)
    
    # Freeze BERT layers (optional - fine-tune for better results)
    # for layer in bert.layers:
    #     layer.trainable = False
    
    # Input layers
    input_ids = Input(shape=(128,), dtype=tf.int32, name='input_ids')
    attention_mask = Input(shape=(128,), dtype=tf.int32, name='attention_mask')
    
    # Get BERT outputs
    outputs = bert(
        input_ids=input_ids,
        attention_mask=attention_mask
    )
    
    # Use [CLS] token or pooling
    cls_output = outputs.last_hidden_state[:, 0, :]  # [CLS] token
    # Or: pooled_output = outputs.pooler_output
    
    # Classification head
    x = Dropout(dropout_rate)(cls_output)
    x = Dense(256, activation='relu')(x)
    x = Dropout(dropout_rate)(x)
    predictions = Dense(num_classes, activation='softmax')(x)
    
    model = Model(inputs=[input_ids, attention_mask], outputs=predictions)
    
    return model

# Build model
model = build_bert_classifier(num_classes=2)
model.compile(
    optimizer=tf.keras.optimizers.Adam(learning_rate=2e-5),
    loss='categorical_crossentropy',
    metrics=['accuracy']
)

# Usage with tokenizer
tokenizer = AutoTokenizer.from_pretrained('bert-base-uncased')

def encode_texts(texts, max_length=128):
    return tokenizer(
        texts,
        padding=True,
        truncation=True,
        max_length=max_length,
        return_tensors='tf'
    )
```

## Optimizers Comparison

### Which Optimizer to Use?

| Optimizer | Best For | Pros | Cons |
|-----------|----------|------|------|
| **Adam** | Default choice | Fast convergence, adaptive lr | Can miss global minima |
| **AdamW** | L2 regularization | Better weight decay | Slower than Adam |
| **SGD** | Research, fine-tuning | Good generalization | Slow, needs tuning |
| **RMSprop** | RNNs, online learning | Handles non-stationary | Sensitive to hyperparameters |
| **Adamax** | Large datasets | Robust to outliers | Less common |

### Adam Optimizer (Recommended Default)


| Scenario | Initial LR | Notes |
|----------|-----------|-------|
| Fine-tuning BERT | 2e-5 to 5e-5 | Very small |
| Transfer learning | 1e-4 to 1e-3 | 10x smaller than from scratch |
| Training from scratch | 1e-3 to 1e-2 | Standard |

## Loss Functions

### Classification Losses

```python
# Binary Crossentropy (2 classes)
loss = 'binary_crossentropy'  # Use with sigmoid output

# Categorical Crossentropy (>2 classes, one-hot encoded)
loss = 'categorical_crossentropy'  # Use with softmax output

# Sparse Categorical Crossentropy (>2 classes, integer labels)
loss = 'sparse_categorical_crossentropy'  # No need for one-hot encoding

# Focal Loss (for imbalanced data)
import tensorflow as tf

def focal_loss(gamma=2.0, alpha=0.25):
    def loss(y_true, y_pred):
        epsilon = tf.keras.backend.epsilon()
        y_pred = tf.clip_by_value(y_pred, epsilon, 1.0 - epsilon)
        
        pt = tf.where(tf.equal(y_true, 1), y_pred, 1 - y_pred)
        alpha_t = tf.where(tf.equal(y_true, 1), alpha, 1 - alpha)
        
        focal = -alpha_t * tf.pow(1 - pt, gamma) * tf.math.log(pt)
        return tf.reduce_mean(focal)
    return loss
```

### Custom Loss Functions

```python
import tensorflow as tf

def custom_mse(y_true, y_pred):
    """MSE with sample weights support"""
    return tf.reduce_mean(tf.square(y_true - y_pred))

def dice_loss(y_true, y_pred, smooth=1e-6):
    """For segmentation tasks"""
    y_true_f = tf.keras.backend.flatten(y_true)
    y_pred_f = tf.keras.backend.flatten(y_pred)
    intersection = tf.reduce_sum(y_true_f * y_pred_f)
    return 1 - (2. * intersection + smooth) / (
        tf.reduce_sum(y_true_f) + tf.reduce_sum(y_pred_f) + smooth
    )

def combined_loss(y_true, y_pred):
    """Combine multiple losses"""
    bce = tf.keras.losses.binary_crossentropy(y_true, y_pred)
    dice = dice_loss(y_true, y_pred)
    return bce + dice
```

## Training Best Practices

### 1. Callbacks for Better Training

```python

        'best_model.keras',

        mode='max',
```

### Cross-Validation for Neural Networks

```python
from sklearn.model_selection import KFold
import numpy as np

def train_with_kfold(X, y, n_splits=5, epochs=50, batch_size=32):
    kfold = KFold(n_splits=n_splits, shuffle=True)
    fold_scores = []
    
    for fold, (train_idx, val_idx) in enumerate(kfold.split(X)):
        X_train, X_val = X[train_idx], X[val_idx]
        y_train, y_val = y[train_idx], y[val_idx]
        
        model = build_model()
        model.fit(
            X_train, y_train,
            validation_data=(X_val, y_val),
            epochs=epochs,
            batch_size=batch_size,
            verbose=0
        )
        
        score = model.evaluate(X_val, y_val, verbose=0)
        fold_scores.append(score[1])  # Accuracy
        
    return fold_scores

# Train
scores = train_with_kfold(X, y, n_splits=5)
print(f"Mean CV Accuracy: {np.mean(scores):.4f} (+/- {np.std(scores):.4f})")
```

## Hyperparameter Tuning

### Key Hyperparameters to Tune

| Hyperparameter | Typical Range | Notes |
|----------------|---------------|-------|
| Learning Rate | 1e-4 to 1e-1 | Most important |
| Batch Size | 16, 32, 64, 128 | Power of 2 |
| Hidden Units | 32 to 512 | Depends on data |
| Dropout | 0.1 to 0.5 | Start with 0.3 |
| Layers | 2 to 10 | Deeper isn't always better |
| Embedding Dim | 50 to 300 | For NLP |
| Attention Heads | 4 to 16 | For Transformers |
